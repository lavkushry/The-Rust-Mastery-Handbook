# Review Checklist

Use this checklist for pull requests affecting handbook content, docs workflows, or repository operations.

## Correctness

- Technical statements are accurate.
- Rust terminology is used correctly.
- Claims are aligned with actual repository capabilities.

## Clarity

- Explanation flow is coherent and easy to follow.
- Reader intent is clear for each section.
- Important takeaways are explicit.

## Internal Links and Structure

- Added/updated links resolve correctly.
- Navigation changes are intentional.
- Cross-links point to the best canonical page.

## Formatting and Readability

- Markdown structure is clean.
- Heading levels are consistent.
- Code fences and command blocks are correctly typed.

## Reader Value

- Change improves comprehension, usability, or contributor workflow.
- Scope is meaningful and not cosmetic churn.

## Scope Control

- PR stays focused on one objective.
- No unrelated changes are mixed in.
- Large structural changes were discussed in an issue first.

## Technical Validation

- mdbook build succeeds.
- Render-sensitive changes were previewed locally.
- Workflow changes were reviewed for permissions and safety.

## Voice Consistency

- Matches practical first-principles handbook tone.
- Avoids hype or generic filler.
- Preserves systems-oriented framing.
