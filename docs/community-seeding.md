# Community Seeding Plan

This plan helps share the project in communities where it naturally fits.

## Where This Project Fits Naturally

- Rust community forums and discussion spaces
- Backend and systems engineering communities
- Documentation and technical-writing communities
- Open-source contributor onboarding communities

## Community-Fit Posting Angles

### Rust-focused communities

Best angle:

- deep mental-model approach to ownership, lifetimes, and contribution readiness

Post format:

- one concise value statement
- one concrete chapter or appendix highlight
- one request for technical feedback

### Backend/systems engineering communities

Best angle:

- transition path for experienced engineers adopting Rust

Post format:

- pain point addressed
- practical outcomes
- low-friction getting-started link

### Docs and technical writing communities

Best angle:

- maintainable mdBook workflow and contributor-first docs architecture

Post format:

- what docs structure worked
- what workflow decisions improved contribution quality

### OSS onboarding communities

Best angle:

- clear first-contribution pathways and maintainer playbooks

Post format:

- one newcomer-friendly issue type
- one concrete onboarding doc link

## What to Avoid

- mass cross-posting identical messages
- exaggerated growth or impact claims
- vague launch posts without technical specifics
- posting without active follow-up in comments

## Healthy Promotion Pattern

1. Share one technically useful post.
2. Answer questions in-thread.
3. Convert useful feedback into issues or docs improvements.
4. Publish a follow-up update only after meaningful progress.
