# Contribution Ladder

This ladder helps contributors choose work that matches their current familiarity.

## Path A: First-Time Contributors

Step 1:
- fix typos, grammar, and broken links in one chapter

Step 2:
- improve section clarity with small wording changes

Step 3:
- open a focused PR with clear before/after explanation

## Path B: Documentation Contributors

Step 1:
- improve FAQ/troubleshooting pages

Step 2:
- add chapter cross-links and consistency fixes

Step 3:
- propose and implement new appendix content

## Path C: Bug-Fix Contributors

Step 1:
- reproduce a reported build or rendering issue

Step 2:
- add a minimal fix with validation notes

Step 3:
- improve workflow robustness in CI or deployment files

## Path D: Feature Contributors

Step 1:
- discuss idea scope in an issue

Step 2:
- align acceptance criteria with maintainers

Step 3:
- implement incrementally and document behavior impact

## Path E: Maintainer Path

Step 1:
- triage and label incoming issues

Step 2:
- review PRs and coach new contributors

Step 3:
- drive release notes and roadmap updates

## Promotion Signals

Contributors are ready for higher-impact work when they consistently:

- ship scoped, reviewable PRs
- respond constructively to review feedback
- validate changes and document tradeoffs
- help others in issues or discussions
