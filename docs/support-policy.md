# Support Policy

This project is maintained on a best-effort basis.

## What Maintainers Support

Maintainers prioritize:

- reproducible bugs in repository content and workflows
- contribution onboarding issues
- security and workflow integrity concerns
- high-impact clarity and structure improvements

## What Is Out of Scope for Maintainer Support

- general Rust tutoring unrelated to repository content
- custom environment debugging without reproducible repository context
- private consulting requests

## Preferred Support Channels

1. GitHub Discussions for usage and open-ended questions
2. GitHub Issues for reproducible bugs and focused feature requests
3. SECURITY.md process for vulnerabilities

Support routing setup references:

- docs/discussions-seed.md
- docs/manual-github-settings-checklist.md

## Response Expectations

Best effort targets:

- first acknowledgment for issues: within 7 days
- follow-up review for actionable PRs: within 14 days

These are targets, not guarantees.

## Closing Policy

Issues may be closed when:

- they are out of scope
- no reproducible details are provided
- requested follow-up details are not provided for a prolonged period
- they are duplicates of tracked issues

When possible, maintainers should close with a brief explanation and guidance.
