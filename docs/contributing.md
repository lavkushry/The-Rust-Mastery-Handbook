# Contributing Guide (Docs View)

For the canonical contributor workflow, read CONTRIBUTING.md.

This page provides a docs-focused quick route.

## Docs-First Contribution Path

1. Pick one scoped improvement.
2. Run mdbook build before opening a PR.
3. Include clear before/after notes in the PR.

## Good First Targets

- typo fixes in one chapter
- broken links
- FAQ and troubleshooting improvements
- glossary clarity edits
